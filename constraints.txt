# Dependency caps for a reproducible Neuron install (pip install -c constraints.txt).
# These do NOT pin exact versions — they cap the majors so a future breaking release
# can't silently land during an install. install.ps1 passes this automatically when
# present; DELETE this file to always take the latest resolvable versions.
#
# Rationale per pin:
#   mcp        — cap the major; the MCP SDK moves fast and 2.x could break the server.
#   fastembed  — it's pre-1.0, and minor bumps have changed embedding behavior
#                (e.g. mean- vs CLS-pooling), which shifts the vector space. Cap tight.
#   pyturso    — pinned in pyproject.toml (authoritative), not duplicated here.
#   fastembed  — pre-1.0; ≥0.8 because 0.5–0.7 pin pillow<12 which has no cp314 wheel
#                (pillow 10.4–11.x can't build from source on Windows without zlib).
#                Dev env already runs 0.8.0 — the pooling-change concern is stale.
mcp>=1.28,<2
fastembed>=0.8,<1
