# Neuron 5.1.0 "Synapse"

Neuron 5.1.0 consolidates the **FiveFix** line of work. It fixes real defects in
how memories are searched and extracted, makes the per-turn path noticeably
cheaper, and — the headline — hardens the persistence layer so a **team can build
one shared knowledge base on Turso Cloud** without corrupting or overwriting each
other's memory.

Upgrading is safe: no data migration, no re-embed, and single-user installs keep
working exactly as before. A few behaviours are deliberately different (search is
now correctly calibrated); those are called out under "Things to know" below.

## Why it matters

Before this release, two things quietly undermined the memory on the local
(non-Turso-SQL) path: vector "similarity" was a raw dot product on non-normalized
vectors, so scores ran far above 1.0 and every relevance threshold was trivially
cleared (everything linked, every association looked "strong"); and vector search
returned on the first non-empty database, so the bundled seed knowledge shadowed
the user's own live graph. Both are fixed — search now uses a true cosine in
[0, 1], consistently across the cloud and local tiers, and it merges the seed with
your active graph instead of hiding it.

At the same time, the heuristic extractor was silently mangling accented Italian
words ("città" → "citt", "perché" → "perch") and leaking those stems as keywords.
Extraction now folds accents before matching, so real nouns survive clean and
accented connectors are filtered.

## Highlights

**Correctness**
- True cosine similarity on the Python fallback tier, consistent with Turso's
  server-side `vector_distance_cos`.
- Vector search merges the seed and the active graph (the seed no longer shadows
  your live memory).
- The `.env` diagnostic checks every `TURSO_` line, not just the first.
- DB toggle in the legacy configuration wrapper now simply flips `#` on `TURSO_*` lines — no
  hardcoded keys, no credential deletion, no re-entry needed.

**Precision (extraction)**
- Accent-folding: accented words are no longer truncated to garbage stems, and
  accented connectors ("più", "perché", "così", "però", "cioè") are filtered.

**Performance**
- Per-(model, text) embedding cache — the same keywords are embedded once per
  turn instead of four or five times.
- The immutable seed database connection is reused for the session instead of
  reopened on every search.

**Concurrency — shared Turso Cloud, multiple writers**
- Per-user session state (turn counter, staged "while-you-were-away" stimulus,
  last-topic) is kept in a **local per-user sidecar** instead of the shared
  `meta` table, so colleagues no longer overwrite each other's turn count or
  receive each other's stimulus.
- **Atomic remote save**: a save is flushed as a single libSQL transaction batch,
  so a colleague loading mid-save never sees a half-written graph.
- **Reconcile guard**: a consolidate/merge that would delete rows another person
  added is downgraded to an additive write on a shared store, unless run as
  coordinated maintenance (`NS_ALLOW_SHARED_RECONCILE=1`).
- **One-shot cloud initializer** (`scripts/init_cloud.py`): migrate the shared
  schema once, up front, so lazy per-client migration never races.
- **Embedding-model write guard + retry/backoff**: a writer whose embedding model
  differs from the store's is prevented from mixing incompatible vectors, and
  transient network blips are retried instead of silently losing a turn.

**Tooling**
- `.gitattributes` normalizes line endings (LF for code/docs, CRLF for
  `.bat`/`.cmd`/`.ps1`) and prevents the CRLF/LF churn from recurring.
- New regression suite `tests/test_fivefix.py` covering every fix above.

## Setting up a shared knowledge base (team)

1. One person runs the one-shot initializer against the shared cloud database,
   before anyone else connects:

   ```
   python scripts/init_cloud.py
   ```

2. Everyone uses the **same** `NS_EMBED_MODEL` (vectors from different models are
   not comparable; a mismatched writer is blocked from writing vectors).

3. Prefer **one context per feature/area** (distinct `NEURON_SLUG` / context) —
   writes stay isolated, while Neuron's cross-context spark and drift links still
   surface associations across features.

4. Don't run `consolidate`/`prune` ad-hoc during shared work. Do it as coordinated
   maintenance, alone, with `NS_ALLOW_SHARED_RECONCILE=1`.

## Things to know (all backward-compatible)

- Search associations will differ from 5.0.x — this is the cosine fix landing;
  the new results are the correctly-calibrated ones.
- On an existing **shared remote** store, each user's `turn_count` restarts from 0
  once (it reads its new, empty local sidecar). Harmless. Local single-user stores
  are unaffected.
- No data migration and no re-embed are required.

## Upgrading

Pull 5.1.0 and re-sync your active install (the MCP server runs the deployed copy,
not the source tree). No store changes are needed.
